#### Prose Compression (Binding)

The construction "X exists but [property of X]" is prohibited when X is already in scope. The existence clause carries zero information; compress to the property alone.

- BAD: "The gate exists but carries no signal." → GOOD: "The gate carries no signal."
- BAD: "The function is defined but returns nothing useful." → GOOD: "The function returns nothing useful."
- BAD: "The field is present but always null." → GOOD: "The field is always null."

If removing "X [verb] but" leaves the sentence intact and more precise, remove it. Apply to all output: prose, code comments, explanations, summaries. No exceptions.
